# 프로젝트 인수인계서 (Handover Document)

본 문서는 서울 지하철 광고 영업 관리 시스템(Lead Manager)의 운영 및 개발 유지보수 업무를 인수하는 담당자를 위한 가이드입니다.

## 1. 프로젝트 개요
- **시스템명**: 서울 지하철 광고 영업 관리 시스템 (Lead Manager)
- **목적**: 지하철 인근 병원/의원 영업 리드 발굴, 관리 및 광고 제안서 자동 생성
- **현재 버전**: v1.0.0 (2026.01.17 기준)

## 3. 주요 인계 사항

### 3.1 계정 및 권한
- **Supabase**: 관리자 계정 접근 권한 및 `service_role` 키 관리 주의 필요.
- **공공데이터포털**: '지방행정인허가데이터' API 활용 신청 연장 필요 여부 확인.
- **Vercel**: 배포 파이프라인 접근 권한 및 환경 변수 접근 권한.

### 3.2 최근 주요 변경 사항
- **리팩토링**: `LeadDetailPanel` 컴포넌트에서 역사 도면(`StationFloorPlans`) 로직 분리 완료.
- **버그 수정**: 리드 데이터 중복 조회/저장 문제 해결 (중복 제거 로직 강화).
- **데이터**: 2026년 1월 기준 서울시 1~9호선 주요 역사 광고 인벤토리 데이터 갱신 완료.

### 3.3 향후 개선 제안 (Backlog)
- 리드 데이터 증가에 따른 서버 사이드 페이지네이션 도입 검토.
- 제안서 발송 이메일 연동 (현재 PDF 다운로드만 지원, `Resend` 등으로 바로 발송 기능 추가).
- 모바일 뷰 최적화 고도화.
