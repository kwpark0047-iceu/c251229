# 테스트 케이스 문서 (Test Cases)

서울 지하철 광고 영업 관리 시스템의 주요 기능에 대한 테스트 시나리오 및 절차를 정의합니다.

## 1. 인증 및 사용자 관리 (Authentication)

| ID | 테스트 항목 | 사전 조건 | 테스트 절차 | 예상 결과 | 비고 |
|---|---|---|---|---|---|
| AUTH-001 | 로그인 성공 | 등록된 계정 존재 | 1. 이메일/비밀번호 입력<br>2. 로그인 버튼 클릭 | 대시보드 메인으로 이동 | |
| AUTH-002 | 로그인 실패 | - | 1. 잘못된 비밀번호 입력<br>2. 로그인 버튼 클릭 | "로그인 실패" 메시지 표시 | |
| AUTH-003 | 로그아웃 | 로그인 상태 | 1. 우측 상단 프로필 클릭<br>2. 로그아웃 선택 | 로그인 페이지로 이동 | |

## 2. 리드 관리 (Lead Manager / CRM)

| ID | 테스트 항목 | 사전 조건 | 테스트 절차 | 예상 결과 | 비고 |
|---|---|---|---|---|---|
| CRM-001 | 리드 데이터 조회 | - | 1. 대시보드 진입<br>2. 데이터 로딩 대기 | 리드 목록이 그리드/리스트에 정상 표시됨 | **중복 데이터가 없어야 함** |
| CRM-002 | 검색 및 필터 | - | 1. 검색창에 "치과" 입력<br>2. 상태 필터 "신규" 선택 | 해당 조건에 맞는 리드만 필터링됨 | |
| CRM-003 | 리드 상태 변경 | 리드 선택 | 1. 상세 패널에서 상태 변경 (New -> Contacted) | DB 업데이트 및 UI에 즉시 반영됨 | |
| CRM-004 | 지도 뷰 확인 | - | 1. 우측 상단 "지도" 탭 클릭 | 지도 위에 마커가 표시되고 클러스터링됨 | |
| CRM-005 | 역사 도면 확인 | 역사 정보 보유 리드 | 1. 리드 상세 패널 하단 스크롤<br>2. 도면 썸네일 클릭 | **전체 화면 모달로 도면이 확대 표시됨** | 최근 리팩토링 항목 |
