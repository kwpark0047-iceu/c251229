# 서울 지하철 광고 영업 시스템 - 요구사항 정의서

**프로젝트명:** 위마켓 (WeMarket)
**버전:** 1.0
**작성일:** 2026-01-12
**문서 유형:** 소프트웨어 요구사항 명세서 (SRS)

---

## 1. 개요

### 1.1 목적
본 문서는 서울 지하철 광고 영업을 위한 리드 관리 시스템의 기능적/비기능적 요구사항을 정의한다.

### 1.2 시스템 개요
지하철 광고 영업팀이 잠재 고객(리드)을 발굴, 관리, 추적하고 광고 인벤토리를 효율적으로 운영할 수 있는 통합 영업 관리 플랫폼이다.

### 1.3 대상 사용자
| 사용자 유형 | 설명 |
|------------|------|
| 영업 담당자 | 리드 발굴, 통화, 제안서 발송, 계약 진행 |
| 영업 관리자 | 팀원 관리, 실적 모니터링, 인벤토리 관리 |
| 시스템 관리자 | 조직 관리, 설정, 데이터 백업 |

### 1.4 용어 정의
| 용어 | 정의 |
|------|------|
| 리드 (Lead) | 광고 영업 대상이 되는 잠재 고객 (사업장) |
| 인벤토리 | 판매 가능한 광고 매체 (포스터, 디지털 사이니지 등) |
| 제안서 | 고객에게 발송하는 광고 상품 제안 문서 |
| CRM | 고객 관계 관리 (Customer Relationship Management) |

## 2. 기능 요구사항

### 2.1 사용자 인증 및 조직 관리

#### FR-AUTH-001: 회원가입
- **설명:** 사용자는 이메일과 비밀번호로 회원가입할 수 있다
- **입력:** 이메일, 비밀번호
- **출력:** 계정 생성 완료, 인증 이메일 발송
- **우선순위:** 필수

#### FR-AUTH-002: 로그인/로그아웃
- **설명:** 사용자는 이메일/비밀번호로 로그인하고 로그아웃할 수 있다
- **입력:** 이메일, 비밀번호
- **출력:** 세션 생성/종료, 대시보드 리다이렉트
- **우선순위:** 필수

#### FR-AUTH-003: 조직 생성
- **설명:** 사용자는 새로운 조직을 생성할 수 있다
- **입력:** 조직명
- **출력:** 조직 생성, 초대 코드 발급
- **우선순위:** 필수

#### FR-AUTH-004: 조직 가입
- **설명:** 사용자는 초대 코드로 기존 조직에 가입할 수 있다
- **입력:** 초대 코드
- **출력:** 조직 멤버 등록
- **우선순위:** 필수

#### FR-AUTH-005: 멤버 역할 관리
- **설명:** 조직 관리자는 멤버의 역할을 변경할 수 있다
- **역할 유형:** owner, admin, member
- **우선순위:** 높음

### 2.2 리드 관리

#### FR-LEAD-001: 리드 자동 수집
- **설명:** LocalData.go.kr API에서 사업자 인허가 데이터를 자동 수집한다
- **입력:**
  - 지역 코드 (서울, 경기, 인천)
  - 업종 카테고리 (건강, 동물, 식품, 문화, 생활, 자원환경, 기타)
  - 날짜 범위 (인허가일 기준)
- **출력:** 리드 목록 (사업장명, 주소, 전화번호, 좌표 등)
- **처리 규칙:**
  - API 호출 시 200ms 지연 (Rate Limiting 방지)
  - 배치 저장 50건씩
  - 중복 체크 (사업장명 + 도로명주소)
- **우선순위:** 필수

#### FR-LEAD-002: 좌표 변환 및 역 매칭
- **설명:** 수집된 좌표를 WGS84로 변환하고 가장 가까운 지하철역을 자동 매칭한다
- **입력:** GRS80 좌표 (EPSG:5174/5181)
- **출력:**
  - WGS84 좌표 (위도, 경도)
  - 최인접 역명
  - 역까지 거리 (미터)
  - 해당 역 노선 목록
- **우선순위:** 필수

#### FR-LEAD-003: 리드 목록 조회
- **설명:** 저장된 리드를 다양한 조건으로 조회할 수 있다
- **필터 조건:**
  - 상태 (신규, 제안발송, 컨택완료, 계약성사)
  - 업종 카테고리
  - 세부 업종 (병원, 의원, 약국 등)
  - 인근 역
  - 날짜 범위
- **정렬:** 인허가일 내림차순
- **우선순위:** 필수

#### FR-LEAD-004: 리드 뷰 모드
- **설명:** 리드를 3가지 뷰 모드로 확인할 수 있다
- **뷰 모드:**
  - 그리드 뷰: 카드 형식
  - 리스트 뷰: 테이블 형식
  - 지도 뷰: Leaflet 지도 마커
- **우선순위:** 필수

#### FR-LEAD-005: 리드 상태 변경
- **설명:** 리드의 영업 상태를 변경할 수 있다
- **상태 흐름:** NEW → PROPOSAL_SENT → CONTACTED → CONTRACTED
- **자동 처리:**
  - CONTACTED 변경 시 현재 사용자를 담당자로 자동 지정
- **우선순위:** 필수

#### FR-LEAD-006: 리드 상세 정보
- **설명:** 리드의 상세 정보를 확인하고 수정할 수 있다
- **표시 정보:**
  - 기본 정보 (사업장명, 주소, 전화, 업종)
  - 위치 정보 (좌표, 인근역, 거리)
  - 통화 기록 목록
  - 제안서 목록
  - 영업 진행 체크리스트
- **우선순위:** 필수

#### FR-LEAD-007: 리드 메모
- **설명:** 리드에 메모를 추가/수정할 수 있다
- **우선순위:** 높음

#### FR-LEAD-008: 리드 담당자 지정
- **설명:** 리드에 담당 영업사원을 지정할 수 있다
- **표시:** 담당자명, 지정 시간
- **우선순위:** 높음

#### FR-LEAD-009: 리드 내보내기
- **설명:** 리드 목록을 CSV 파일로 내보낼 수 있다
- **우선순위:** 중간

#### FR-LEAD-010: 중복 리드 제거
- **설명:** 중복된 리드를 자동으로 탐지하고 삭제할 수 있다
- **중복 기준:** 사업장명 + 도로명주소
- **우선순위:** 중간

### 2.3 광고 인벤토리 관리

#### FR-INV-001: 인벤토리 조회
- **설명:** 광고 인벤토리 목록을 조회할 수 있다
- **필터 조건:**
  - 역명
  - 광고 유형
  - 가용 상태
- **우선순위:** 필수

#### FR-INV-002: 인벤토리 등록
- **설명:** 새로운 광고 인벤토리를 등록할 수 있다
- **입력 항목:**
  - 역명 (필수)
  - 위치 코드 (필수)
  - 광고 유형 (포스터, 디지털사이니지, 랩핑, 기둥, 에스컬레이터, 바닥, 스크린도어, 라이트박스)
  - 광고 크기
  - 월간/주간 가격
  - 가용 상태 (가용, 예약, 사용중)
  - 이용 가능 기간
  - 일 평균 유동인구
  - 타겟 인구통계
  - 설명
- **우선순위:** 필수

#### FR-INV-003: 인벤토리 수정/삭제
- **설명:** 기존 인벤토리 정보를 수정하거나 삭제할 수 있다
- **우선순위:** 필수

#### FR-INV-004: 엑셀 일괄 업로드
- **설명:** Excel 파일로 인벤토리를 일괄 등록할 수 있다
- **지원 형식:** .xlsx, .xls
- **필드 매핑:** 역명, 위치코드, 광고유형, 크기, 가격, 상태, 설명
- **처리:**
  - 드래그 & 드롭 지원
  - 진행 상황 표시
  - 오류 행 표시
- **우선순위:** 높음

#### FR-INV-005: 인벤토리-도면 연동
- **설명:** 인벤토리를 도면상의 위치와 연동할 수 있다
- **입력:** X, Y 좌표 (0-100%)
- **우선순위:** 중간

### 2.4 제안서 관리

#### FR-PROP-001: 제안서 작성
- **설명:** 리드에 대한 광고 제안서를 작성할 수 있다
- **입력 항목:**
  - 제목
  - 인사말
  - 광고 인벤토리 선택 (복수)
  - 할인율
- **자동 계산:**
  - 총 가격
  - 할인 적용 최종 가격
- **우선순위:** 필수

#### FR-PROP-002: 제안서 상태 관리
- **설명:** 제안서의 상태를 관리한다
- **상태:** 작성중(DRAFT) → 발송됨(SENT) → 열람됨(VIEWED) → 수락됨(ACCEPTED) / 거절됨(REJECTED)
- **우선순위:** 필수

#### FR-PROP-003: 이메일 발송
- **설명:** 작성된 제안서를 이메일로 발송할 수 있다
- **이메일 내용:**
  - HTML 템플릿
  - 노선 배지 (색상 코딩)
  - 광고 항목 테이블
  - 예상 효과 분석
- **자동 처리:**
  - 제안서 상태: SENT
  - 리드 상태: PROPOSAL_SENT
- **우선순위:** 필수

#### FR-PROP-004: PDF 생성
- **설명:** 제안서를 PDF 파일로 생성할 수 있다
- **우선순위:** 높음

#### FR-PROP-005: AI 자동 제안
- **설명:** AI가 고객 정보 기반으로 맞춤형 광고 상품을 추천한다
- **입력:** 고객명, 회사, 연락처, 주소, 업종, 예산
- **출력:**
  - 인근 역 추천 (유동인구 기반)
  - 추천 광고 유형
  - 예산 기반 구성안
- **우선순위:** 높음

#### FR-CRM-001: 통화 기록 등록

  - 메모
  - 다음 조치
  - 콜백 요청일
- **자동 처리:**
  - 관심/미팅잡힘 시 리드 상태 CONTACTED로 변경
- **우선순위:** 필수

#### FR-CRM-002: 통화 기록 조회
- **설명:** 리드별 통화 기록 이력을 조회할 수 있다
- **정렬:** 통화일시 내림차순
- **우선순위:** 필수

#### FR-CRM-003: 영업 진행 체크리스트

- **표시:** 각 단계 완료일, 메모

#### FR-CRM-004: 콜백 알림
- **설명:** 콜백 요청일에 알림을 표시한다
- **우선순위:** 중간

### 2.6 업무 스케줄 관리

### 2.9 통계 및 대시보드

## 5. 외부 인터페이스 요구사항

### 5.1 LocalData.go.kr API
- **목적:** 사업자 인허가 데이터 조회
- **방식:** REST API (XML 응답)
- **인증:** API 키 (서버사이드)
- **주요 파라미터:**
  - opnSvcId: 서비스 ID
  - localCode: 지역 코드
  - pageIndex, pageSize: 페이지 정보
  - lastModTsBgn, lastModTsEnd: 날짜 범위

### 5.2 KRIC API
- **목적:** 지하철역 편의시설 정보 조회
- **방식:** REST API
- **인증:** API 키 (서버사이드)

### 5.3 Resend API
- **목적:** 이메일 발송
- **방식:** REST API
- **인증:** API 키 (서버사이드)

### 5.4 Supabase
- **목적:** 데이터베이스, 인증, 파일 저장
- **구성요소:**
  - PostgreSQL 데이터베이스
  - Auth (인증)
  - Storage (파일 저장)
  - RLS (Row Level Security)
